## ${en:'Related projects', ru:'Похожие проекты'}

Other tools in the same space — useful if ${name} does not fit your stack or workflow:<!--en-->
Другие инструменты в этой области — пригодятся, если ${name} не подходит под ваш стек или рабочий процесс:<!--ru-->

- **[multilingual-markdown](https://github.com/ryul1206/multilingual-markdown)** (Python) — generates per-language markdown files from a single base document,<!--en-->
  similar to NRG's multi-language flow but without widgets or template imports.<!--en-->
- **[multilingual-markdown](https://github.com/ryul1206/multilingual-markdown)** (Python) — генерирует markdown-файлы на разных языках из одного базового документа,<!--ru-->
  аналогично многоязычному режиму NRG, но без виджетов и импортов шаблонов.<!--ru-->
- **[embedme](https://github.com/zakhenry/embedme)** (Node.js) — embeds external code snippets into markdown via fenced-block annotations;<!--en-->
  overlaps with NRG's `import` widget (region / lines extraction).<!--en-->
- **[embedme](https://github.com/zakhenry/embedme)** (Node.js) — встраивает внешние фрагменты кода в markdown через аннотации code-блоков;<!--ru-->
  пересекается с виджетом `import` NRG (извлечение по регионам и диапазонам строк).<!--ru-->
- **[cog](https://github.com/nedbat/cog)** (Python) — runs embedded Python snippets inside source files to generate text;<!--en-->
  a different philosophy (code execution instead of declarative widgets) for the same goal of <!--en-->
  keeping generated documentation in sync with its source of truth.<!--en-->
- **[cog](https://github.com/nedbat/cog)** (Python) — исполняет встроенные Python-сниппеты прямо в исходных файлах для генерации текста;<!--ru-->
  другая философия (исполнение кода вместо декларативных виджетов), но та же цель — <!--ru-->
  поддерживать сгенерированную документацию в актуальном состоянии.<!--ru-->
- **[doctoc](https://github.com/thlorenz/doctoc)** (Node.js) — auto-generates and updates the table of contents in an existing markdown file;<!--en-->
  a focused alternative to NRG's `tableOfContents` widget.<!--en-->
- **[doctoc](https://github.com/thlorenz/doctoc)** (Node.js) — автоматически генерирует и обновляет оглавление в существующем markdown-файле;<!--ru-->
  узконаправленная альтернатива виджету `tableOfContents` в NRG.<!--ru-->
- **[remark](https://github.com/remarkjs/remark)** (Node.js) — pluggable markdown processor; with plugins such as `remark-toc` and `remark-include`<!--en-->
  it can be composed into a pipeline that covers several NRG features separately.<!--en-->
- **[remark](https://github.com/remarkjs/remark)** (Node.js) — расширяемый markdown-процессор; с плагинами вроде `remark-toc` и `remark-include`<!--ru-->
  его можно собрать в конвейер, покрывающий по частям несколько возможностей NRG.<!--ru-->
